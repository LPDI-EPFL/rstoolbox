rstoolbox.components.DesignFrame.make_resfile¶

DesignFrame.make_resfile(seqID, header, filename, write=True)¶

Generate a Rosetta resfile to match the design’s sequence assuming the reference_sequence as the starting point.

The function picks the mutant positions between the design sequence and the reference_sequence and generates a resfile that will transform one into the other.

If more than one design is provided, the resfile name (filename) is going to be modified with a counter. A new column will be added to the data container in order to keep track of the filename assignation:

New Column Data Content
resfile_<seqID> Name of the resfile
Parameters:
  • seqID (str) – Identifier of the sequence of interest.
  • header (str) – Header content for the resfile; defines default behaviour.
  • filename (str) – Identifier of the resfile. Will be altered with a numerical suffix if the data container holds more thant one sequence.
  • write (bool) – Testing attribute. When False, resfiles are not actually created
Returns:

Union[DesignSeries, DesignFrame] - Itself with the new column.
Raise:
NotImplementedError:
 if the data container is not DataFrame or Series.
KeyError:if there is no reference_sequence for chain seqID of the decoys
IOError:if output file exists and system.overwrite is False

Note

Depends on system.overwrite and system.output.
